# Baby Step 3 — Full Symptômes section

## Context

Baby Steps 1 and 2 are complete. The app has:
- React + Vite + TypeScript + Tailwind CSS, deployed on GitHub Pages
- Bottom navigation with 4 tabs (tab switching works)
- `src/screens/AujourdhuiScreen.tsx` — all 6 section cards interactive, persisted to IndexedDB
- `src/db.ts` — IndexedDB wrapper with `JournalEntry` type and `getEntry` / `saveEntry`
- The Symptômes section only has one field: "Migraine aujourd'hui ?" (Oui/Non toggle)
- The Repas card is a disabled placeholder

The goal of this step is to expand the **Symptômes section** with all the fields from the spec. Everything else stays untouched.

---

## What to implement

### 1. Extend `JournalEntry` in `src/db.ts`

Add the missing symptom fields to the interface:

```ts
migraineAuReveil?: boolean
heureApparitionMigraine?: string   // "HH:MM"
heureDisparitionMigraine?: string  // "HH:MM"
intensiteMigraine?: number         // 0–10
localisationMigraine?: string[]
nausees?: boolean
migraineSoulageeParRepas?: boolean
```

No migration needed — IndexedDB `put` is an upsert, old entries simply won't have the new fields.

---

### 2. Update `src/screens/AujourdhuiScreen.tsx`

#### New state variables

Add these alongside the existing ones:

```ts
const [migraineAuReveil, setMigraineAuReveil] = useState<boolean | null>(null)
const [heureApparitionMigraine, setHeureApparitionMigraine] = useState<string>('')
const [heureDisparitionMigraine, setHeureDisparitionMigraine] = useState<string>('')
const [intensiteMigraine, setIntensiteMigraine] = useState<number>(5)
const [localisationMigraine, setLocalisationMigraine] = useState<string[]>([])
const [nausees, setNausees] = useState<boolean | null>(null)
const [migraineSoulageeParRepas, setMigraineSoulageeParRepas] = useState<boolean | null>(null)
```

#### Load on mount

Populate the new fields from the loaded entry, same pattern as existing fields.

#### Symptômes section — new layout

When `migraine === false` or `migraine === null`: show only the "Migraine aujourd'hui ?" toggle (unchanged).

When `migraine === true`: show the toggle **plus** the following fields in order:

**"Présente dès le réveil ?"** — Oui/Non toggle → `migraineAuReveil`

**"Heure d'apparition"** — `<input type="time">` → `heureApparitionMigraine`
- Visible only when `migraineAuReveil === false`

**"Heure de disparition"** — `<input type="time">` → `heureDisparitionMigraine`
- Always visible when `migraine === true` (optional field)

**"Intensité"** — `<input type="range" min="0" max="10" step="1">` → `intensiteMigraine`
- Display the current value next to the label: *"Intensité : 7 / 10"*
- Default value: 5

**"Localisation"** — multi-select using pill buttons → `localisationMigraine`
- Options: `["Front", "Tempes", "Derrière les yeux", "Nuque", "Diffus", "Autre"]`
- Each pill toggles its value in/out of the array (independent, not mutually exclusive)
- Selected pill style: background `var(--color-primary)`, text white
- Unselected pill style: background transparent, border `var(--color-border)`, text `var(--color-text-muted)`
- Pills wrap in a flex row

**"Nausées ?"** — Oui/Non toggle → `nausees`

**"Soulagée par un repas ?"** — Oui/Non toggle → `migraineSoulageeParRepas`

#### Save button

Include all new fields in the saved `JournalEntry`. For `boolean | null` fields, include only if not null. For `intensiteMigraine`, `localisationMigraine`, `heureApparitionMigraine`, `heureDisparitionMigraine`: include only if `migraine === true`. Include `intensiteMigraine` only if `migraine === true` (do not save a default 5 when there's no migraine).

---

## Oui/Non toggle behavior (unchanged from Step 2)

Selecting the already-active option deselects it (sets back to `null`).
Selecting the inactive option sets it. Mutually exclusive.

**Active style:** background `var(--color-primary)`, text white.
**Inactive style:** background transparent, border `var(--color-border)`, text `var(--color-text-muted)`.
