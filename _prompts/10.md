# Baby Step 10 — PWA Update Banner

## Context

Baby Steps 1–9 are complete. The app has:
- React + Vite + TypeScript + Tailwind CSS, deployed on GitHub Pages as a PWA
- Bottom navigation with 4 tabs
- `src/App.tsx` — root component, renders the active screen + `BottomNav`
- `vite.config.ts` — uses `vite-plugin-pwa` with `registerType: 'autoUpdate'`

The current `autoUpdate` strategy silently installs new service worker versions in the background but keeps serving old cached assets until the user manually hard-reloads. Users on Android with the installed PWA never see updates.

The goal of this step is to show a non-blocking banner when a new version is available, so users can update with one tap.

---

## What to implement

### 1. Update `vite.config.ts`

Change `registerType` from `'autoUpdate'` to `'prompt'`:

```ts
VitePWA({
  registerType: 'prompt',
  // rest unchanged
})
```

With `prompt`, the new service worker waits instead of activating automatically. The app controls when to apply the update.

---

### 2. Update `src/App.tsx`

Add the `useRegisterSW` hook from `virtual:pwa-register/react` (provided by `vite-plugin-pwa`, no new dependency needed).

```ts
import { useRegisterSW } from 'virtual:pwa-register/react'
```

Inside `App`, call the hook:

```ts
const { needRefresh: [needRefresh], updateServiceWorker } = useRegisterSW()
```

When `needRefresh` is `true`, render an update banner **between the opening `<div>` and `<main>`** (i.e. above the screen content, below nothing — it pushes the content down):

**Banner layout:**
- Full-width strip, background `var(--color-primary)`, white text.
- Left side: text `"Une mise à jour est disponible"`, small font.
- Right side: button labelled `"Mettre à jour"`, white background, text `var(--color-primary)`, small font, rounded, compact padding.
- On button click: call `updateServiceWorker(true)`. This sends `SKIP_WAITING` to the waiting service worker and reloads the page.

When `needRefresh` is `false` (the common case), render nothing — no empty space.

---

## Success criteria

- `npm run build` completes without errors, no TypeScript errors.
- `npm run dev`: the banner does not appear during normal use (no waiting SW in dev).
- After deploying a new build to GitHub Pages: users who already have the app open see the banner at the top. Tapping "Mettre à jour" reloads the app with the new version.
- All other screens and behavior are unchanged.

---

## What NOT to do

- Only change `vite.config.ts` and `src/App.tsx`.
- No new npm dependencies.
- No test files.
- Do not add a dismiss/close button — the only action is to update.
- Do not add the update check inside `ParametresScreen` — the banner must be visible on all tabs.
