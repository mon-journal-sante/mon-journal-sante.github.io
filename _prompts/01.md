# Baby Step 1 — Project scaffold + PWA shell deployable on GitHub Pages

## Context

This is a French PWA (Progressive Web App) for tracking migraines and health factors, targeting Android. The repo follows the `<org>.github.io` convention, so GitHub Pages serves the app at the root path `/`.

The goal of this step is to produce a **deployable PWA shell** with the correct structure, visual theme, and navigation — no data logic, no API calls.

## Stack

- React + Vite + TypeScript
- Tailwind CSS
- `vite-plugin-pwa`
- Hosting: GitHub Pages
- CI/CD: GitHub Actions

## What to implement

### 1. Initialize the project

Run `npm create vite@latest . -- --template react-ts` in the current directory (the repo root), then install dependencies:

```
npm install
npm install -D tailwindcss @tailwindcss/vite
npm install vite-plugin-pwa
```

### 2. Configure Tailwind CSS

Use the Vite plugin approach (Tailwind v4+). Add `@import "tailwindcss"` to the main CSS file.

Define these custom CSS variables (used throughout the app):

```css
:root {
  --color-bg: #F9F8FF;
  --color-surface: #FFFFFF;
  --color-primary: #8B7CF8;
  --color-primary-dark: #7C6CF6;
  --color-text: #18181B;
  --color-text-muted: #6B7280;
  --color-border: #EDE9FE;
  --color-success: #10B981;
  --color-danger: #EF4444;
}
```

Set `background-color: var(--color-bg)` on `body` and `min-height: 100dvh`.

### 3. Configure Vite + PWA

`vite.config.ts`:

```ts
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'
import { VitePWA } from 'vite-plugin-pwa'

export default defineConfig({
  base: '/',
  plugins: [
    react(),
    tailwindcss(),
    VitePWA({
      registerType: 'autoUpdate',
      manifest: {
        name: 'Mon Journal Santé',
        short_name: 'Journal',
        lang: 'fr',
        theme_color: '#8B7CF8',
        background_color: '#F9F8FF',
        display: 'standalone',
        orientation: 'portrait',
        icons: [
          { src: '/icon-192.png', sizes: '192x192', type: 'image/png' },
          { src: '/icon-512.png', sizes: '512x512', type: 'image/png' }
        ]
      }
    })
  ]
})
```

### 4. PWA icon assets

Create `public/icon.svg`:

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 512 512">
  <rect width="512" height="512" rx="100" fill="#8B7CF8"/>
  <rect x="148" y="128" width="220" height="272" rx="16" fill="white" opacity="0.95"/>
  <rect x="148" y="128" width="30" height="272" rx="10" fill="#7C6CF6"/>
  <polyline
    points="200,268 222,268 242,210 264,322 284,232 302,284 318,268 346,268"
    fill="none" stroke="#8B7CF8" stroke-width="12"
    stroke-linecap="round" stroke-linejoin="round"/>
</svg>
```

For `icon-192.png` and `icon-512.png`: generate them from the SVG using the `sharp` npm package (install as a dev dependency, write a one-off script `scripts/generate-icons.ts`, run it once, then the PNG files should be committed to `public/`).

### 5. GitHub Actions deployment

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy to GitHub Pages
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      pages: write
      id-token: write
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npm run build
      - uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist
      - id: deployment
        uses: actions/deploy-pages@v4
```

### 6. App structure

Keep it simple — no router library. Use a single `useState` to track which tab is active.

`src/App.tsx` renders:
- `<main>` — the active screen
- `<BottomNav>` — fixed bottom navigation

Screens (one component each, in `src/screens/`):
- `AujourdhuiScreen` — the main screen (see section 7)
- `HistoriqueScreen` — empty placeholder: centred text *"Historique à venir"*
- `CorrelationsScreen` — empty placeholder: centred text *"Corrélations à venir"*
- `ParametresScreen` — empty placeholder: centred text *"Paramètres à venir"*

### 7. Bottom navigation

`src/components/BottomNav.tsx`

Fixed at the bottom. Four tabs:

| Icon (text emoji is fine as placeholder) | Label | Screen key |
|---|---|---|
| 📅 | Aujourd'hui | `aujourdhui` |
| 📖 | Historique | `historique` |
| 📊 | Corrélations | `correlations` |
| ⚙️ | Paramètres | `parametres` |

Active tab: text color `var(--color-primary)`. Inactive: `var(--color-text-muted)`.
Background: `var(--color-surface)`. Top border: `1px solid var(--color-border)`.

### 8. Écran "Aujourd'hui" — structure only

`src/screens/AujourdhuiScreen.tsx`

Header: *"Aujourd'hui"* + today's date formatted in French (e.g. *"dimanche 26 avril 2026"*).

Six section cards, displayed in order. Each card: white background (`var(--color-surface)`), rounded corners, subtle border (`var(--color-border)`), section title in bold.

**Section 1 — Sommeil**
Title: "Sommeil 🌙"
Contents (static layout, no logic):
- "Heure de coucher" — time input
- "Heure de lever" — time input
- "Durée : —" (static placeholder)
- "Fenêtre ouverte cette nuit ?" — two buttons Oui / Non (no state)
- "Symptômes dès le réveil ?" — two buttons Oui / Non (no state)

**Section 2 — Repas**
Title: "Repas 🍽️"
Contents:
- Button "+ Ajouter un repas" (no action yet)

**Section 3 — Hydratation**
Title: "Hydratation 💧"
Contents:
- "Verres d'eau : 0" with − and + buttons (no state)

**Section 4 — Symptômes**
Title: "Symptômes"
Contents:
- "Migraine aujourd'hui ?" — two buttons Oui / Non (no state)

**Section 5 — Cycle menstruel**
Title: "Cycle menstruel"
Contents:
- "Jour de règles ?" — two buttons Oui / Non (no state)

**Section 6 — Notes libres**
Title: "Notes libres 📝"
Contents:
- `<textarea>` placeholder *"Stress, médicament, activité physique…"* (no state)

Footer: a full-width button "Enregistrer la journée" in `var(--color-primary)` color (no action yet).

The screen scrolls vertically. Bottom padding ensures the last card is not hidden behind the bottom nav.

## Success criteria

- `npm run build` completes without errors
- `npm run dev` shows the app locally: bottom nav works, switching tabs shows each screen, "Aujourd'hui" sections are visible and scroll correctly
- The GitHub Actions workflow file is valid YAML
- PWA manifest is included in the built `dist/`
- No TypeScript errors

## What NOT to do

- No data storage (IndexedDB, localStorage) — that's a later step
- No Claude API integration
- No notification logic
- No real interactivity in the form fields (Oui/Non buttons and inputs are rendered but carry no state)
- No routing library (React Router etc.) — a simple `useState` is enough for 4 tabs
- No test files
