# Baby Step 6 — Photo capture + Claude API food identification

## Context

Baby Steps 1–5 are complete. The app has:
- React + Vite + TypeScript + Tailwind CSS, deployed on GitHub Pages
- Bottom navigation with 4 tabs
- `src/screens/AujourdhuiScreen.tsx` — all 6 sections fully interactive, persisted to IndexedDB
- `src/db.ts` — IndexedDB wrapper with `JournalEntry` (including `Repas[]`), `getEntry`, `saveEntry`, `clearAllEntries`
- `src/screens/ParametresScreen.tsx` — Claude API key stored in `localStorage` under key `claude_api_key`
- The Repas card has a working inline add form (category, time, aliments tag input, note) — but no photo capture

The goal of this step is to add **photo capture and automatic food identification via the Claude API** inside the Repas add form. Everything else stays untouched.

---

## What to implement

### 1. Create `src/utils/claude.ts`

This module handles the Claude API call. It must not import anything from React.

```ts
export async function identifyFoodFromPhoto(
  base64Image: string,
  mimeType: string,
  apiKey: string
): Promise<string[]>
```

- Calls `https://api.anthropic.com/v1/messages` with method `POST`.
- Headers:
  - `Content-Type: application/json`
  - `x-api-key: <apiKey>`
  - `anthropic-version: 2023-06-01`
  - `anthropic-dangerous-direct-browser-access: true`
- Body:
  ```json
  {
    "model": "claude-haiku-4-5-20251001",
    "max_tokens": 512,
    "messages": [
      {
        "role": "user",
        "content": [
          {
            "type": "image",
            "source": {
              "type": "base64",
              "media_type": "<mimeType>",
              "data": "<base64Image>"
            }
          },
          {
            "type": "text",
            "text": "Tu es un assistant nutritionnel. Identifie tous les aliments visibles dans cette photo de repas. Réponds uniquement en JSON avec un tableau `aliments` contenant les noms en français. Sois précis et exhaustif."
          }
        ]
      }
    ]
  }
  ```
- On success: parse the response JSON, extract `content[0].text`, parse that as JSON, return `data.aliments` (a `string[]`).
- If parsing fails or `aliments` is missing: throw an `Error`.
- If the HTTP response is not ok: throw an `Error` with the status code.

### 2. Update the inline add form in `src/screens/AujourdhuiScreen.tsx`

No changes outside the Repas inline add form.

#### New local state inside the form

Add alongside the existing form state:

```ts
const [photoLoading, setPhotoLoading] = useState<boolean>(false)
const [photoError, setPhotoError] = useState<string | null>(null)
```

#### Photo button — placement

Add a **"📷 Identifier par photo"** button just above the aliments tag input row.

- Style: secondary (border `var(--color-border)`, text `var(--color-text-muted)`, transparent bg, full width, rounded).
- This button is a label wrapping a hidden `<input type="file" accept="image/*" capture="environment">`.
  - Use `useRef` to hold the input ref. The button click triggers `inputRef.current?.click()`.
  - The `<input>` is hidden (`display: none`).
- When `photoLoading === true`: replace the button text with `"Identification en cours…"` and disable the button.

#### On photo selected (`onChange` of the file input)

1. Get the selected file. If none, return.
2. Read the API key from `localStorage.getItem('claude_api_key')`.
3. If the API key is empty or missing:
   - Set `photoError` to `"Clé API manquante. Renseignez-la dans les Paramètres."`.
   - Return early.
4. Convert the file to base64 using `FileReader` (read as Data URL, then strip the `data:...;base64,` prefix).
5. Set `photoLoading = true`, clear `photoError`.
6. Call `identifyFoodFromPhoto(base64, file.type, apiKey)`.
7. On success: **append** the returned aliments to `formAliments` (deduplicate: skip names already in the list, case-insensitive). Set `photoLoading = false`.
8. On error: set `photoError` to `"Identification échouée. Ajoutez les aliments manuellement."`, set `photoLoading = false`.

#### Error display

When `photoError` is not null: show a small line of text in `var(--color-danger)` directly below the photo button. It disappears as soon as the user successfully adds an aliment manually or triggers a new photo.

## Success criteria

- `npm run build` completes without errors, no TypeScript errors.
- `npm run dev`:
  - The "📷 Identifier par photo" button appears above the aliments input in the add form.
  - Without an API key in localStorage: clicking the button shows the "Clé API manquante" error message. No API call is made.
  - With a valid API key: selecting a photo shows "Identification en cours…", then populates the aliments list with the identified foods, deduplicating against already-added items.
  - If the API call fails: the error message appears and the aliments list is unchanged.
  - The manual aliment tag input still works as before (unaffected by this step).
  - "Ajouter" remains disabled until at least one aliment is present (unchanged).
  - Saving then refreshing restores meals including those populated via photo.
- All other sections and screens are unchanged.
