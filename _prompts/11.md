### 4. The 7 correlations

#### Corr. 1 — Aliments fréquents les jours de migraine

**Card title:** `"🍽️ Aliments les jours de migraine"`

**Computation:**
- Take all entries where `migraine === true`.
- Flatten all `repas[*].aliments` into one list.
- Count occurrences per aliment (case-insensitive; keep first-seen casing for display).
- Keep only aliments that appear **≥ 2 times**. Sort descending by count.

**Display:**
- If fewer than 3 migraine entries: *"Pas encore assez de données (il faut au moins 3 jours de migraine)."*
- If no aliment reaches 2 occurrences: *"Pas encore de récurrence dans les aliments les jours de migraine."*
- Otherwise, top 5 at most, comma-separated:
  > *"Pain (5 fois), Fromage (3 fois), Œufs (2 fois)"*

#### Corr. 3 — Sommeil et migraine

**Card title:** `"🌙 Sommeil et migraine"`

**Computation:**
- For each entry that has both `heureCoucher` and `heureLever`, compute sleep duration in minutes:
  parse each as `H*60+M`; if `leverMin ≤ coucherMin` add 1440; result = `leverMin − coucherMin`.
- Split entries into: those with `migraine === true` (and valid sleep data) vs. `migraine === false` (and valid sleep data).
- Compute average duration in minutes for each group. Convert to `"Xh YY"` format for display (pad minutes with leading zero if < 10).

**Display:**
- If either group has fewer than 3 entries: *"Pas encore assez de données."*
- Otherwise:
  > *"Nuits avant une migraine : 6h15 de sommeil en moyenne."*
  > *"Nuits sans migraine : 7h45 de sommeil en moyenne."*

#### Corr. 6 — Cycle menstruel

**Card title:** `"🌸 Cycle menstruel"`

---

**Step 1 — Resolve `debutDernierCycle` and read cycle settings**

Read cycle parameters from localStorage (all optional, with medically grounded defaults):

```ts
const settings = JSON.parse(localStorage.getItem('settings') ?? '{}')
const dureeCycle: number        = settings.dureeCycle        ?? 28  // total cycle length, valid range 21–35
const dureeLuteale: number      = settings.dureeLuteale      ?? 14  // luteal phase length, valid range 10–17
const dureeMenstruation: number = settings.dureeMenstruation ?? 5   // menstruation length, valid range 3–7
```

These fields will be set by the Paramètres screen in a future step. Until then, defaults apply.

Then resolve `debutDernierCycle` (ISO date `"YYYY-MM-DD"` of cycle day 1) by trying two sources in order:

**Source 1 — Explicit setting:**

```ts
let debutDernierCycle: string | undefined = settings.debutDernierCycle
```

**Source 2 — Auto-inferred from `jourRegles` entries (used only if Source 1 is undefined):**

If `debutDernierCycle` is still undefined, scan the journal entries for runs of consecutive days where `jourRegles === true`:

1. Take all entries where `jourRegles === true` and sort them by date ascending.
2. Walk through them and group into runs: a new run starts whenever the gap between two adjacent entries is ≥ 2 days.
3. Take the most recent run. Its **first day** (earliest date in that run) becomes the inferred `debutDernierCycle`.

If no `jourRegles === true` entry exists at all, `debutDernierCycle` remains `undefined` — handled by Fallback A below.

---

**Step 2 — Validate parameters**

Before any computation, apply two checks:

**Check A — cycle length in range:**
```
dureeCycle >= 21 AND dureeCycle <= 35
```
If this fails: the cycle length is outside the medically normal range (polyménorrhée / oligoménorrhée).

**Check B — follicular phase has at least 1 day (from `cycle-menstruel-specifications.md` §4.3):**
```
jourOvulation    = dureeCycle − dureeLuteale
debutOvulatoire  = jourOvulation − 1
(dureeMenstruation + 1) <= (debutOvulatoire − 1)
```
If this fails: the menstruation duration leaves no room for a follicular phase — the parameters are incoherent.

If either check fails, show:
> *"Paramètres de cycle incohérents. Vérifie la durée du cycle et la durée des règles dans les Paramètres."*

Do not proceed further for this card.

---

**Step 3 — Compute phase boundaries**

Using the validated parameters, derive all boundaries once:

```
jourOvulation    = dureeCycle − dureeLuteale
debutOvulatoire  = jourOvulation − 1
finOvulatoire    = jourOvulation + 1

Phase menstruelle  : jours 1                   → dureeMenstruation
Phase folliculaire : jours dureeMenstruation+1  → debutOvulatoire−1
Phase ovulatoire   : jours debutOvulatoire      → finOvulatoire
Phase lutéale      : jours finOvulatoire+1      → dureeCycle
```

Example with defaults (28-day cycle, 5-day menstruation, 14-day luteal):
- jourOvulation = 14, debutOvulatoire = 13, finOvulatoire = 15
- Menstruelle: j. 1–5 · Folliculaire: j. 6–12 · Ovulatoire: j. 13–15 · Lutéale: j. 16–28

---

**Step 4 — Phase resolution helper**

Implement `getPhase()` exactly as specified in `cycle-menstruel-specifications.md` §7:

```ts
type Phase = 'menstruelle' | 'folliculaire' | 'ovulatoire' | 'luteale'

function getPhase(
  jourCourant: number,
  dureeCycle: number,
  dureeLuteale: number,
  dureeMenstruation: number
): Phase | null {
  if (jourCourant < 1 || jourCourant > dureeCycle) return null
  const jourOvulation   = dureeCycle - dureeLuteale
  const debutOvulatoire = jourOvulation - 1
  const finOvulatoire   = jourOvulation + 1
  if (jourCourant <= dureeMenstruation) return 'menstruelle'
  if (jourCourant <  debutOvulatoire)   return 'folliculaire'
  if (jourCourant <= finOvulatoire)     return 'ovulatoire'
  return 'luteale'
}
```

Returns `null` for a day out of the valid cycle range — treat such entries as unclassifiable and skip them.

---

**Step 5 — In-memory enrichment of `jourCycle`**

Some entries already have `jourCycle` (manually entered). For those that don't, compute it from `debutDernierCycle` if it is set:

```
elapsed = floor((Date.parse(entry.date) − Date.parse(debutDernierCycle)) / 86400000)
```

- If `elapsed < 0`: the entry predates the declared cycle start — skip, leave `jourCycle` undefined.
- If `elapsed >= 0`: `computedJourCycle = (elapsed % dureeCycle) + 1`

Produce a local enriched array — **do not write back to IndexedDB**. Entries that already have a manual `jourCycle` are used as-is.

---

**Step 6 — Computation**

Using the enriched entries, call `getPhase(entry.jourCycle, dureeCycle, dureeLuteale, dureeMenstruation)` for each entry that has a defined `jourCycle`. Entries where `getPhase()` returns `null` are skipped.

For each of the 4 phases, compute:
- `total` — count of enriched entries classified into that phase (regardless of `migraine` value)
- `migraines` — of those, count where `migraine === true`

---

**Step 7 — Display**

**Fallback A** — `debutDernierCycle` could not be resolved from either source AND no entry has a manual `jourCycle`:
> *"Pour activer cette corrélation, renseigne la date de début de ton dernier cycle dans les Paramètres, ou commence à cocher 'Jour de règles ?' dans ton journal quotidien."*

**Fallback B** — parameters are incoherent (Step 2 failed):
> *"Paramètres de cycle incohérents. Vérifie la durée du cycle et la durée des règles dans les Paramètres."*

**Fallback C** — fewer than 3 enriched entries have a defined, classifiable `jourCycle`:
> *"Pas encore assez de données sur le cycle."*

**Main display** — show one line per phase that has `total > 0`, in fixed order (menstruelle → folliculaire → ovulatoire → lutéale). Each line shows the phase name, its hormonal context in muted text, and the migraine count:

| Phase | Libellé | Contexte hormonal affiché (muted) |
|---|---|---|
| `menstruelle` | "Phase menstruelle" | "chute des œstrogènes" |
| `folliculaire` | "Phase folliculaire" | "montée des œstrogènes" |
| `ovulatoire` | "Phase ovulatoire" | "pic de LH" |
| `luteale` | "Phase lutéale" | "chute de la progestérone" |

Line format:
> *"Phase [nom] ([contexte]) : X migraine(s) sur Y jours"*

Example:
> *"Phase menstruelle (chute des œstrogènes) : 3 migraines sur 5 jours"*
> *"Phase folliculaire (montée des œstrogènes) : 1 migraine sur 8 jours"*
> *"Phase ovulatoire (pic de LH) : 2 migraines sur 3 jours"*
> *"Phase lutéale (chute de la progestérone) : 4 migraines sur 12 jours"*

**Summary line** — after the phase lines, if exactly one phase has a strictly higher `migraines` count than all other phases with `total > 0`, and that count is ≥ 2, add one summary line in `var(--color-primary)`:
> *"Tes migraines se concentrent en phase [nom] (j. X–Y)."*

Where `X` and `Y` are the computed first and last day of that phase (derived from the boundaries calculated in Step 3). Example: *"Tes migraines se concentrent en phase lutéale (j. 16–28)."*
