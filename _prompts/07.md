# Baby Step 7 — Écran Historique (liste + vue détaillée)

## Context

Baby Steps 1–6 are complete. The app has:
- React + Vite + TypeScript + Tailwind CSS, deployed on GitHub Pages
- Bottom navigation with 4 tabs
- `src/screens/AujourdhuiScreen.tsx` — all 6 sections fully interactive, persisted to IndexedDB
- `src/db.ts` — IndexedDB wrapper with `JournalEntry` (including `Repas[]`), `getEntry`, `saveEntry`, `clearAllEntries`
- `src/screens/ParametresScreen.tsx` — Claude API key (localStorage) + clear all data
- `src/utils/claude.ts` — food identification via the Claude API
- `src/utils/date.ts` — `todayISO()` and `formatDateFR(iso)`
- `src/screens/HistoriqueScreen.tsx` — currently a placeholder ("Historique à venir")

The goal of this step is to implement the **Historique screen**: a list of all recorded days (newest first), with a detail view accessible by tapping a row. No editing in this step.

---

## What to implement

### 1. Add `getAllEntries` to `src/db.ts`

```ts
export async function getAllEntries(): Promise<JournalEntry[]>
```

- Uses `db.getAll(STORE)`.
- Returns entries sorted by `date` descending (newest first).

---

### 2. Rewrite `src/screens/HistoriqueScreen.tsx`

No other file is modified.

#### Local state

```ts
const [entries, setEntries] = useState<JournalEntry[]>([])
const [selected, setSelected] = useState<string | null>(null) // ISO date of the open entry
```

#### Load on mount

Call `getAllEntries()` and store the result in `entries`.

---

#### View: list (when `selected === null`)

If `entries` is empty, show a centered message:
> *"Aucune entrée pour l'instant. Commence à remplir ton journal !"*
> (color: `var(--color-text-muted)`)

Otherwise render a scrollable list. Each row is a tappable card (same Card pattern as the other screens — define `Card` locally in this file, do not import it from elsewhere). Tapping a row sets `selected` to that entry's date.

**Row layout** (flex, space between left content and right icons):

Left side:
- Date formatted with `formatDateFR(entry.date)`, capitalized first letter, bold.
- Below the date: number of meals if any — e.g. `"3 repas"` — in small text, `var(--color-text-muted)`.

Right side (flex row, gap):
- Migraine icon: `✅` if `entry.migraine === false`, `🤕` if `entry.migraine === true`, nothing if `entry.migraine` is undefined.
- Nausées icon: `🤢` if `entry.nausees === true`, nothing otherwise.
- Intensity badge: if `entry.migraine === true` and `entry.intensiteMigraine !== undefined`, show a small badge with the value (e.g. `"7/10"`), background `var(--color-danger)` at 10% opacity, text color `var(--color-danger)`, rounded, small font.

Thin separator line (`var(--color-border)`) between rows.

---

#### View: detail (when `selected` is set)

Find `entry = entries.find(e => e.date === selected)`. If not found, reset `selected` to `null`.

**Header row** (flex, space between):
- Left: `"← Retour"` button — secondary style (no border, text `var(--color-primary)`, transparent bg) — sets `selected` to `null`.
- Center: date formatted with `formatDateFR(entry.date)`, bold, capitalized.
- Right: empty (for layout balance — a visually invisible placeholder of the same size as the back button).

Then display the entry data as a series of read-only cards (same `Card` style), one per section. Only render a card if that section has at least one non-default value (skip entirely empty sections).

**Card: Sommeil** — shown if any of `heureCoucher`, `heureLever`, `fenetreOuverte`, `symptolesReveil` is defined.
- "Coucher : HH:MM" / "Lever : HH:MM"
- If both are defined: "Durée : Xh YY" (compute duration in minutes from the two HH:MM strings)
- "Fenêtre ouverte" or "Fenêtre fermée" if `fenetreOuverte` is defined
- "Symptômes au réveil" if `symptolesReveil === true`

**Card: Repas** — shown if `entry.repas.length > 0`.
- One row per meal: bold category label + time, then aliments as comma-separated text, then note (italic) if present.
- Use this local mapping:
  ```ts
  const CATEGORIE_LABELS: Record<Repas['categorie'], string> = {
    'petit-dejeuner': 'Petit-déjeuner',
    'dejeuner': 'Déjeuner',
    'diner': 'Dîner',
    'collation': 'Collation',
  }
  ```
- Thin separator between meals.

**Card: Hydratation** — shown if `entry.nbVerresEau > 0`.
- "💧 X verre(s) d'eau"

**Card: Symptômes** — shown if `entry.migraine !== undefined`.
- "Migraine : Oui" or "Migraine : Non"
- If `migraine === true`:
  - "Dès le réveil" or "Apparition : HH:MM" (whichever applies)
  - "Disparition : HH:MM" if defined
  - "Intensité : X/10" if defined
  - "Localisation : …" (comma-separated) if defined and non-empty
  - "Nausées : Oui" / "Nausées : Non" if `nausees` is defined
  - "Soulagée par un repas : Oui/Non" if `migraineSoulageeParRepas` is defined

**Card: Cycle** — shown if `entry.jourRegles !== undefined`.
- "Jour de règles : Oui" or "Non"
- "Jour du cycle : X" if `jourCycle` is defined

**Card: Notes** — shown if `entry.notesLibres` is a non-empty string.
- Render the text as-is (preserve newlines with `white-space: pre-wrap`).

## Success criteria

- `npm run build` completes without errors, no TypeScript errors.
- `npm run dev`:
  - With no saved entries: Historique tab shows the empty state message.
  - After saving one or more entries in Aujourd'hui: the Historique list shows them newest first, with the correct date, icons, and intensity badge.
  - Tapping a row opens the detail view with all non-empty sections rendered.
  - "← Retour" goes back to the list.
  - Navigating away and back to Historique re-fetches entries from IndexedDB (list stays up to date).
- All other screens and tabs are unchanged.
