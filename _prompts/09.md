## What to implement

### 1. Create `src/utils/exportCsv.ts`

This module is pure TypeScript — no React imports.

#### Column order and mapping

The CSV has exactly these 20 columns, in this order:

| Column header | Source |
|---|---|
| `date` | `entry.date` |
| `heure_coucher` | `entry.heureCoucher ?? ''` |
| `heure_lever` | `entry.heureLever ?? ''` |
| `duree_sommeil_min` | computed (see below), or `''` |
| `fenetre_ouverte` | `true→'Oui'`, `false→'Non'`, `undefined→''` |
| `repas_categories` | categories of all meals joined with `\|` (e.g. `"petit-dejeuner\|dejeuner"`) |
| `aliments_identifies` | all aliments from all meals joined with `\|` (e.g. `"pain\|beurre\|salade"`) |
| `nb_verres_eau` | `entry.nbVerresEau` (number, always present) |
| `migraine` | `true→'Oui'`, `false→'Non'`, `undefined→''` |
| `migraine_au_reveil` | `true→'Oui'`, `false→'Non'`, `undefined→''` |
| `heure_apparition_migraine` | `entry.heureApparitionMigraine ?? ''` |
| `heure_disparition_migraine` | `entry.heureDisparitionMigraine ?? ''` |
| `intensite_migraine` | `entry.intensiteMigraine ?? ''` |
| `localisation_migraine` | localisation values joined with `\|`, or `''` |
| `nausees` | `true→'Oui'`, `false→'Non'`, `undefined→''` |
| `migraine_soulagee_par_repas` | `true→'Oui'`, `false→'Non'`, `undefined→''` |
| `jour_regles` | `true→'Oui'`, `false→'Non'`, `undefined→''` |
| `jour_cycle` | always `''` (field not yet in data model) |
| `temperature` | `entry.temperature ?? ''` |
| `notes_libres` | `entry.notesLibres ?? ''` |

#### Duration calculation

When both `heureCoucher` and `heureLever` are defined:
1. Parse each as minutes from midnight (`"HH:MM"` → `H*60+M`).
2. If `leverMin < coucherMin`, add `1440` to `leverMin` (slept past midnight).
3. Result = `leverMin - coucherMin`.

If either field is missing, the `duree_sommeil_min` cell is empty.

#### CSV escaping

Write a small helper `escapeCsv(value: string): string`:
- If `value` contains `;`, `"`, or a newline: wrap in double quotes and replace every `"` inside with `""`.
- Otherwise return the value as-is.

Apply this helper to every cell value.

#### Export function

```ts
export async function exportCsv(entries: JournalEntry[]): Promise<void>
```

1. Build the header row: column names joined with `;`.
2. For each entry (in the order provided — caller controls sort), build one data row: map each column, apply `escapeCsv`, join with `;`.
3. Join all rows (header + data) with `\r\n`.
4. Prepend the UTF-8 BOM (`'﻿'`) so Excel français opens it correctly.
5. Create a `Blob` with type `'text/csv;charset=utf-8;'`.
6. Trigger a download:
   - Create a temporary `<a>` element.
   - Set `href` to `URL.createObjectURL(blob)`.
   - Set `download` to `journal-migraines-${todayISO()}.csv` (import `todayISO` from `../utils/date`).
   - Append to `document.body`, click, then remove and call `URL.revokeObjectURL`.

### 2. Update `src/screens/ParametresScreen.tsx`

Add an export button in the Paramètres screen, in its own section card, placed **above** the "Effacer toutes les données" section.

**Section card** (same Card style as the rest of the screen):

- Title: `"Export"`
- A full-width button labelled `"Exporter mes données (CSV)"`.
  - Style: primary (`var(--color-primary)`, white text, rounded).
  - On click: call `getAllEntries()` then pass the result to `exportCsv()`.
  - While the export is running: disable the button and change the label to `"Export en cours…"`.
  - After the download is triggered: re-enable the button (restore original label).

No toast or confirmation needed — the browser's native download is the feedback.

## Success criteria

- `npm run build` completes without errors, no TypeScript errors.
- `npm run dev`:
  - The "Exporter mes données (CSV)" button appears in the Paramètres screen.
  - Clicking it with no saved entries downloads a CSV with only the header row.
  - Clicking it with saved entries downloads a correctly formatted CSV:
    - Semicolon-separated columns.
    - UTF-8 BOM present (open in Excel to verify: accents render correctly).
    - One row per entry, newest-first order (same as `getAllEntries`).
    - Boolean fields show `Oui`/`Non`/empty correctly.
    - Multi-value fields (`repas_categories`, `aliments_identifies`, `localisation_migraine`) joined with `|`.
    - `notes_libres` with embedded newlines or semicolons are properly quoted.
    - Filename is `journal-migraines-YYYY-MM-DD.csv` with today's date.
  - All other screens and behavior are unchanged.
