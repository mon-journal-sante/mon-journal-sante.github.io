# Baby Step 4 — Paramètres screen: API key + clear data

## Context

Baby Steps 1–3 are complete. The app has:
- React + Vite + TypeScript + Tailwind CSS, deployed on GitHub Pages
- Bottom navigation with 4 tabs
- `src/screens/AujourdhuiScreen.tsx` — all 6 sections fully interactive and persisted to IndexedDB
- `src/db.ts` — IndexedDB wrapper with `JournalEntry`, `getEntry`, `saveEntry`
- `src/screens/ParametresScreen.tsx` — currently a placeholder ("Paramètres à venir")

The goal of this step is to implement the **Paramètres screen** with two features:
1. **Claude API key** — stored in `localStorage`, read by the Repas section in a later step
2. **Clear all data** — wipe all IndexedDB entries

Everything else (notifications, cycle date, CSV export) stays for later steps.

---

## What to implement

### 1. Add `clearAllEntries` to `src/db.ts`

```ts
export async function clearAllEntries(): Promise<void>
```

Uses `db.clear(STORE)`.

---

### 2. Rewrite `src/screens/ParametresScreen.tsx`

#### State

```ts
const [apiKey, setApiKey] = useState<string>('')
const [showKey, setShowKey] = useState<boolean>(false)
const [keySaved, setKeySaved] = useState<boolean>(false)
const [confirmClear, setConfirmClear] = useState<boolean>(false)
const [cleared, setCleared] = useState<boolean>(false)
```

#### Load on mount

Read `localStorage.getItem('claude_api_key') ?? ''` into `apiKey`.

#### Layout

Use the same `Card` component pattern as `AujourdhuiScreen` — copy the `Card` helper locally into `ParametresScreen.tsx`. Do not extract it to a shared file yet.

---

#### Section "Clé API Claude"

Card title: `"Clé API Claude 🔑"`

Contents in order:

1. A short paragraph:
   > "Pour activer l'identification automatique des aliments par photo, renseignez votre clé API Claude. Elle est stockée uniquement sur cet appareil."

2. A link: `"Obtenir une clé API → console.anthropic.com"`
   - `href="https://console.anthropic.com"`, `target="_blank"`, `rel="noopener noreferrer"`
   - Small text, color `var(--color-primary)`

3. Input row — full width, flex, items aligned:
   - `<input type={showKey ? 'text' : 'password'}>` bound to `apiKey`
   - Placeholder: `"sk-ant-..."`
   - Flex-grow, same border/bg style as the textarea in `AujourdhuiScreen`
   - To the right: a small button "Afficher" / "Masquer" that toggles `showKey`

4. Full-width "Enregistrer la clé" button:
   - On click: `localStorage.setItem('claude_api_key', apiKey.trim())`, `setKeySaved(true)`, auto-reset to `false` after 2 s
   - When `keySaved === true`: button text = `"Clé enregistrée ✓"`, background `var(--color-success)`
   - Otherwise: primary style (background `var(--color-primary)`, white text)

---

#### Section "Données"

Card title: `"Données"`

Default state (when `confirmClear === false` and `cleared === false`):
- One full-width button "Effacer toutes les données":
  - `border: 1px solid var(--color-danger)`, text color `var(--color-danger)`, transparent background
  - On click: `setConfirmClear(true)` — do NOT delete anything yet

When `confirmClear === true`:
- Replace the button with a confirmation zone:
  - Text: `"Cette action est irréversible. Toutes les entrées du journal seront supprimées."`
  - Two buttons side by side:
    - "Annuler" (secondary style) → `setConfirmClear(false)`
    - "Confirmer la suppression" (background `var(--color-danger)`, white text) → call `clearAllEntries()`, `setConfirmClear(false)`, `setCleared(true)`, auto-reset `cleared` to `false` after 3 s

When `cleared === true`:
- Show a small line of text `"Données effacées."` in `var(--color-text-muted)` where the button was.

## Success criteria

- `npm run build` completes without errors, no TypeScript errors.
- `npm run dev`: navigating to the Paramètres tab shows the new screen with both cards.
- Entering an API key and clicking "Enregistrer la clé" saves it to `localStorage` and shows the green confirmation for ~2 s.
- After page refresh, going to Paramètres shows the previously saved key (masked by default).
- "Afficher" / "Masquer" toggle works.
- Clicking "Effacer toutes les données" shows the confirmation zone.
- "Annuler" returns to the original button state.
- "Confirmer la suppression" calls `clearAllEntries`, shows "Données effacées." for ~3 s, then returns to the original button.
- Other tabs (Aujourd'hui, Historique, Corrélations) are unchanged.
