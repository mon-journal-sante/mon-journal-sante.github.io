# Baby Step 2 — Interactive "Aujourd'hui" + persistence (IndexedDB)

## Context

Baby Step 1 is complete and deployed. The app has:
- React + Vite + TypeScript + Tailwind CSS, deployed on GitHub Pages
- Bottom navigation with 4 tabs (tab switching works)
- `src/screens/AujourdhuiScreen.tsx` — all 6 section cards rendered, but **no state, no storage**
- 3 placeholder screens (Historique, Corrélations, Paramètres)

The goal of this step is to make the "Aujourd'hui" screen **fully interactive and persistent**:
every field carries real state, saving writes to IndexedDB, reopening the app restores today's data.

The Repas section (add meal, photo, Claude API) stays as a placeholder — that is a later step.

### 2. Database module — `src/db.ts`

Create a typed IndexedDB wrapper using `idb`.

- DB name: `journal-sante`, version `1`
- Object store: `entries`, `keyPath: 'date'`
- Key type: `string` (ISO date `YYYY-MM-DD`)

Use the following entry type (partial — repas will be extended in a later step):

```ts
export interface JournalEntry {
  date: string

  // Sommeil
  heureCoucher?: string        // "HH:MM"
  heureLever?: string          // "HH:MM"
  fenetreOuverte?: boolean
  symptolesReveil?: boolean

  // Repas — kept as empty array for now, extended later
  repas: []

  // Hydratation
  nbVerresEau: number

  // Symptômes (minimal — full symptom detail is a later step)
  migraine?: boolean

  // Cycle
  jourRegles?: boolean

  // Notes
  notesLibres?: string
}
```

Export two functions:

```ts
export async function getEntry(date: string): Promise<JournalEntry | undefined>
export async function saveEntry(entry: JournalEntry): Promise<void>
```

`saveEntry` uses `put` (upsert — creates or overwrites).

### 4. Sleep duration calculation — `src/utils/sleep.ts`

```ts
export function computeDuration(coucher: string, lever: string): string | null
```

- Returns `null` if either input is empty.
- Handles overnight: if `lever` ≤ `coucher` in minutes-since-midnight, add 24 hours to lever.
- Returns formatted string: `"7h30"` if minutes > 0, `"7h00"` if exactly on the hour.
- Cap at 24h (return `null` if result ≥ 24h — malformed input).

### 5. Rewrite `src/screens/AujourdhuiScreen.tsx`

Replace the static layout with a fully stateful version. Keep the same visual structure — only add state and behavior.

#### State

```ts
const [heureCoucher, setHeureCoucher] = useState<string>('')
const [heureLever, setHeureLever] = useState<string>('')
const [fenetreOuverte, setFenetreOuverte] = useState<boolean | null>(null)
const [symptolesReveil, setSymptolesReveil] = useState<boolean | null>(null)
const [nbVerresEau, setNbVerresEau] = useState<number>(0)
const [migraine, setMigraine] = useState<boolean | null>(null)
const [jourRegles, setJourRegles] = useState<boolean | null>(null)
const [notesLibres, setNotesLibres] = useState<string>('')
const [toast, setToast] = useState<boolean>(false)
```

`boolean | null` means: `null` = not yet answered, `true` = Oui, `false` = Non.

#### Load on mount

On mount, call `getEntry(todayISO())`. If an entry exists, populate all state fields from it.

#### Sommeil section

- `heureCoucher` and `heureLever` are controlled `<input type="time">`.
- Sleep duration: call `computeDuration(heureCoucher, heureLever)`.
  - If result is not null → display `"Durée : 7h30"`.
  - Otherwise → display `"Durée : —"`.
- "Fenêtre ouverte cette nuit ?" → `fenetreOuverte` state, Oui/Non toggle.
- "Symptômes dès le réveil ?" → `symptolesReveil` state, Oui/Non toggle.

#### Hydratation section

- Counter is functional: − decrements (minimum 0), + increments.

#### Symptômes section

- "Migraine aujourd'hui ?" → `migraine` state, Oui/Non toggle.
- No additional fields in this step (full symptom detail is a later step).

#### Cycle menstruel section

- "Jour de règles ?" → `jourRegles` state, Oui/Non toggle.

#### Notes libres section

- Controlled `<textarea>` bound to `notesLibres`.

#### Oui/Non toggle behavior

Selecting the already-active option deselects it (sets back to `null`).
Selecting the inactive option sets it. The two buttons are mutually exclusive.

**Active style** (selected): background `var(--color-primary)`, text white.
**Inactive style** (not selected): background transparent, border `var(--color-border)`, text `var(--color-text-muted)`.

Both buttons always visible; no option is hidden.

#### Save button

On click:
1. Build a `JournalEntry` object from current state. `boolean | null` fields: include only if not `null` (use `?? undefined` or omit). `nbVerresEau` always included. `repas: []`.
2. Call `saveEntry(entry)`.
3. Show the toast for 2.5 seconds (`setTimeout` sets `toast` back to `false`).

#### Toast

A fixed element overlaid above the bottom nav (e.g. `bottom: 80px`, centered horizontally).
Text: `"Journée enregistrée ✓"`.
Background: `var(--color-success)`, text white, rounded, padded.
Only visible when `toast === true`.

## Success criteria

- `npm run build` completes without errors, no TypeScript errors.
- `npm run dev`: opening the app shows the "Aujourd'hui" screen with all fields interactive.
- Oui/Non toggles highlight the selected option; clicking the same option again deselects it.
- Hydration counter increments and decrements; cannot go below 0.
- Filling both sleep times shows the calculated duration.
- Clicking "Enregistrer la journée" shows the green toast for ~2.5s, then it disappears.
- After saving and refreshing the page, all filled fields are restored from IndexedDB.
- The Repas button is visually disabled (no action on click).
- Other tabs (Historique, Corrélations, Paramètres) still show their placeholders unchanged.
