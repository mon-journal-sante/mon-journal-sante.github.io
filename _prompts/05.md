# Baby Step 5 — Repas section (form + persistence, no photo/API)

## Context

Baby Steps 1–4 are complete. The app has:
- React + Vite + TypeScript + Tailwind CSS, deployed on GitHub Pages
- Bottom navigation with 4 tabs
- `src/screens/AujourdhuiScreen.tsx` — all 6 sections fully interactive, persisted to IndexedDB
- `src/db.ts` — IndexedDB wrapper with `JournalEntry`, `getEntry`, `saveEntry`, `clearAllEntries`
- `src/screens/ParametresScreen.tsx` — API key (localStorage) + clear all data
- The Repas card in `AujourdhuiScreen` is currently a disabled placeholder button

In `src/db.ts`, the `JournalEntry` interface has `repas: []` (untyped empty array). In `handleSave` inside `AujourdhuiScreen.tsx`, repas is hardcoded to `[]`.

The goal of this step is to make the **Repas section fully functional**: add meals, see the list, delete a meal, persist to IndexedDB. No photo capture, no Claude API call — those are Step 6.

---

## What to implement

### 1. Add `Repas` type and update `JournalEntry` in `src/db.ts`

Add this interface **above** `JournalEntry`:

```ts
export interface Repas {
  id: string                  // uuid — use crypto.randomUUID()
  categorie: 'petit-dejeuner' | 'dejeuner' | 'diner' | 'collation'
  heure: string               // "HH:MM"
  aliments: string[]
  note?: string
}
```

Update the `repas` field in `JournalEntry`:

```ts
repas: Repas[]
```

No DB migration needed — IndexedDB `put` is an upsert; old entries with `repas: []` stay valid.

### 2. Update `src/screens/AujourdhuiScreen.tsx`

#### New state

Add alongside the existing state:

```ts
const [repas, setRepas] = useState<Repas[]>([])
const [showRepasForm, setShowRepasForm] = useState<boolean>(false)
```

#### Load on mount

Populate `repas` from the loaded entry:

```ts
if (entry.repas?.length) setRepas(entry.repas)
```

#### Update `handleSave`

Replace the hardcoded `repas: []` with the actual state:

```ts
repas,
```

#### Repas card — new layout

Replace the disabled placeholder button with:

1. **List of added meals** — one summary row per meal (see below). Empty if no meals yet.
2. A full-width **"+ Ajouter un repas"** button (primary style: `var(--color-primary)`, white text, rounded).
   - When `showRepasForm === false`: clicking it sets `showRepasForm = true`.
   - When `showRepasForm === true`: the button is hidden, the add form is shown instead.

#### Inline add form

Shown directly inside the Repas card, below the meal list, when `showRepasForm === true`.

The form is a local controlled component with its own state (not lifted to the screen level):

```ts
const [formCategorie, setFormCategorie] = useState<Repas['categorie']>('dejeuner')
const [formHeure, setFormHeure] = useState<string>(currentTimeHHMM())
const [formAliments, setFormAliments] = useState<string[]>([])
const [formAlimentInput, setFormAlimentInput] = useState<string>('')
const [formNote, setFormNote] = useState<string>('')
```

Where `currentTimeHHMM()` is a small inline helper that returns the current time as `"HH:MM"`.

Form fields in order:

**Catégorie** — `<select>` with options:
- `petit-dejeuner` → "Petit-déjeuner"
- `dejeuner` → "Déjeuner"
- `diner` → "Dîner"
- `collation` → "Collation"

**Heure** — `<input type="time">` pre-filled with the current time.

**Aliments** — tag input:
- A row with a text `<input>` (placeholder: "Ajouter un aliment…") and a small "+" button.
- Pressing Enter in the input or clicking "+" adds the trimmed value as a tag (ignore empty strings).
- Added aliments displayed as pills below the input: each pill shows the name and an "×" button that removes it.

**Note** — `<textarea>` (2 rows), placeholder: "Remarque optionnelle…", optional.

Two action buttons, side by side:
- **"Annuler"** — secondary style (border `var(--color-border)`, text `var(--color-text-muted)`, transparent bg) → hides the form, resets all form fields.
- **"Ajouter"** — primary style (`var(--color-primary)`, white text) → appends a new `Repas` object to `repas` state (with `id: crypto.randomUUID()`), hides the form, resets all form fields.

"Ajouter" is disabled (visually greyed out, `disabled` attribute) when `formAliments` is empty — at least one food item is required.

#### Meal summary rows

Each saved meal is displayed as a compact row inside the Repas card, above the add button. Layout:

- Left side (flex column):
  - **Category label** (translated, bold, small) + **time** in `var(--color-text-muted)` — on one line
  - Food list: comma-separated aliment names, small text, `var(--color-text-muted)`
  - Note (if present): italic, small, `var(--color-text-muted)`
- Right side: a small "Supprimer" button (text color `var(--color-danger)`, no border, transparent bg, small font)

Clicking "Supprimer" removes that meal from the `repas` array (filter by `id`).

A thin separator line (`var(--color-border)`) between meal rows when there are multiple meals.

## Success criteria

- `npm run build` completes without errors, no TypeScript errors.
- `npm run dev`:
  - The Repas card shows the "+ Ajouter un repas" button (enabled, not disabled).
  - Clicking it shows the inline add form with all fields.
  - Selecting a category, entering a time, adding at least one aliment, then clicking "Ajouter" adds a meal summary row and hides the form.
  - "Ajouter" is disabled when the aliment list is empty.
  - "Annuler" hides the form and discards the draft.
  - Adding multiple meals shows multiple rows in order.
  - "Supprimer" on a meal row removes it immediately.
  - Clicking "Enregistrer la journée" then refreshing the page restores the meal list from IndexedDB.
- All other sections (Sommeil, Hydratation, Symptômes, Cycle, Notes) are unchanged.
- Other tabs (Historique, Corrélations, Paramètres) are unchanged.
