# Baby Step 14 — Saisir un jour passé

## Context

Baby Steps 1–13 are complete. The app has:
- `src/screens/AujourdhuiScreen.tsx` — accepts `date?: string` and `onSaved?: () => void` props. When `date` is provided, it loads and saves data for that specific date. When `onSaved` is provided, the title becomes "Modifier" and the save button becomes "Enregistrer les modifications".
- `src/screens/HistoriqueScreen.tsx` — shows the list of past entries. Clicking an entry opens a `DetailView`, which has a "Modifier" button that renders `AujourdhuiScreen` with `date` and `onSaved` for that entry's date.
- `src/utils/date.ts` — exports `todayISO()` (returns `"YYYY-MM-DD"` for today).

The `HistoriqueScreen` currently only allows editing **existing** entries. There is no way to input data for a past day that has no entry yet.

---

## What to implement

### Only modify `src/screens/HistoriqueScreen.tsx`

Add two new state variables alongside the existing ones:

```ts
const [pastDayMode, setPastDayMode] = useState(false)
const [pastDayDate, setPastDayDate] = useState('')
```

---

### Routing logic (insert before existing routing)

Add a new case at the top of the render logic, before the existing `if (selected && selectedEntry && editing)` check:

```ts
if (pastDayMode && pastDayDate) {
  return (
    <AujourdhuiScreen
      date={pastDayDate}
      onSaved={() => {
        getAllEntries().then(setEntries)
        setPastDayMode(false)
        setPastDayDate('')
      }}
    />
  )
}
```

---

### List view — add the "Saisir un jour passé" UI

In the list view (the final `return`), between the `<h1>Historique</h1>` and the entries list/empty state, add:

**When `!pastDayMode`:** a button `"+ Saisir un jour passé"` that sets `setPastDayMode(true)`. Style: full width, bordered, `var(--color-primary)` border and text, transparent background, same border-radius and padding as other secondary buttons in the app.

**When `pastDayMode && !pastDayDate`:** an inline section with:
- A `<input type="date">` with `max` set to yesterday (`todayISO()` minus 1 day — compute inline as `new Date(Date.now() - 86400000).toISOString().slice(0, 10)`). On `onChange`, call `setPastDayDate(e.target.value)` — this immediately triggers the routing above.
- A small "Annuler" link/button below (or beside) the date input that calls `setPastDayMode(false)`.
- Style the date input the same as other inputs in the app (`rounded-lg border px-2 py-1.5 text-sm`, `var(--color-border)`, `var(--color-text)`, `var(--color-bg)`).

---

## Success criteria

- `npm run build` completes without errors, no TypeScript errors.
- `npm run dev`:
  - In the Historique list view, a "Saisir un jour passé" button is visible.
  - Clicking it shows a date input (max = yesterday) and an "Annuler" option.
  - Clicking "Annuler" hides the date input and shows the button again.
  - Selecting a past date immediately opens the full journal form (same as Aujourd'hui / edit form) for that date.
  - If the date has no entry, the form is empty. If it already has an entry, the form is pre-filled.
  - After saving, the Historique list is refreshed and the user is back on the list view.
  - All existing Historique behaviors (list, detail view, edit) are unchanged.
