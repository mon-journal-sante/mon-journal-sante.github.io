Additionally, **Fallback A** (line 219) references "les Paramètres", which is a broken promise since no such settings exist in the UI.

### Why `DUREE_LUTEALE = 14` is a true constant

The luteal phase is biologically stable regardless of cycle length. Its duration (14 days) is determined by the corpus luteum lifespan, not by observable behavior. Detecting it requires BBT tracking or LH tests — neither of which this app records. Per `cycle-menstruel-specifications.md` §2.1:

> *"La phase lutéale est relativement stable quelle que soit la durée du cycle. Toute variation de durée du cycle se répercute sur la phase folliculaire, jamais sur la phase lutéale."*

`DUREE_LUTEALE = 14` stays hardcoded. It is never derived from journal data.

### What CAN be derived from `jourRegles` journal data

The `jourRegles === true` entries, grouped into consecutive runs, give us:

- **`DUREE_MENSTRUATION`** — the average length of those runs (how many days each period lasts). Available from the first cycle of data.
- **`DUREE_CYCLE`** — the average gap in days between consecutive run start dates (period start to period start). Requires at least 2 runs. Fallback to 28 if fewer than 2 runs exist.
- **`debutDernierCycle`** — the start date of the most recent run (already used by Source 2, unchanged).

#### Step 1 — Run detection (replaces current Source 2 and the settings read)

Build a structured array of runs from `jourRegles === true` entries:

```ts
const reglesEnts = entries
  .filter(e => e.jourRegles === true)
  .map(e => e.date)
  .sort()

const runs: { start: string; length: number }[] = []
if (reglesEnts.length > 0) {
  let runStart = reglesEnts[0]
  let runLen = 1
  for (let i = 1; i < reglesEnts.length; i++) {
    const gap = (Date.parse(reglesEnts[i]) - Date.parse(reglesEnts[i - 1])) / 86400000
    if (gap >= 2) {
      runs.push({ start: runStart, length: runLen })
      runStart = reglesEnts[i]
      runLen = 1
    } else {
      runLen++
    }
  }
  runs.push({ start: runStart, length: runLen })
}
```

A new run starts whenever the gap between two adjacent `jourRegles` entries is **≥ 2 days**. Consecutive days (gap = 1) continue the same run.

---

#### Step 2 — Derive all cycle values from runs

```ts
const DUREE_LUTEALE = 14  // true biological constant — never derived

const DUREE_MENSTRUATION = runs.length > 0
  ? Math.round(runs.reduce((sum, r) => sum + r.length, 0) / runs.length)
  : 5

const DUREE_CYCLE = runs.length >= 2
  ? Math.round(
      runs.slice(1).reduce(
        (sum, r, i) => sum + (Date.parse(r.start) - Date.parse(runs[i].start)) / 86400000,
        0
      ) / (runs.length - 1)
    )
  : 28

const debutDernierCycle: string | undefined =
  runs.length > 0 ? runs[runs.length - 1].start : undefined
```

**`DUREE_CYCLE` formula explained:** `runs.slice(1)` iterates over every run except the first. At index `i`, `runs[i]` is the preceding run. The gap = `Date.parse(current.start) - Date.parse(previous.start)` in days. Average all gaps, round to nearest integer.
